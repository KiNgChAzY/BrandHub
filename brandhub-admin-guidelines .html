Primary Logo
The primary logo is the cornerstone of our brand identity. Use the full lockup — symbol + wordmark — wherever possible.
Alternate Logos
Approved alternate lockups for specific contexts where the primary logo cannot be used.
Icon Mark
Use the standalone icon for small applications like favicons, app icons, and social media avatars where the full lockup won't read clearly.
Usage Rules
✓ Do
  • Use on white or light backgrounds
  • Maintain clear space equal to the cap height of the "A"
  • Use approved colour variants only
  • Scale proportionally at all times
✗ Don't
  • Stretch or distort the logo in any direction
  • Place on busy or clashing backgrounds
  • Recreate, redraw, or alter the logo
  • Add effects like drop shadows or outlines
Clear Space & Minimum Size
Always maintain a minimum clear space around the logo equal to the cap height of the wordmark "A". This ensures the logo breathes and reads clearly in any context.

Primary Palette
These are the core colours of the Acme brand. Use them consistently across all digital and print touchpoints.
Secondary Palette
Supplementary colours for data visualisation, status indicators, and accent moments.
Tints & Shades
Use tints (lighter) for backgrounds and containers. Use shades (darker) sparingly for depth and emphasis.
Colour Usage Rules
✓ Do
  • Use Acme Blue as the primary CTA and interactive colour
  • Use Deep Navy for headlines on light backgrounds
  • Reserve semantic colours (green, amber, red) for system states
  • Ensure 4.5:1 contrast ratio for text on coloured backgrounds
✗ Don't
  • Mix more than 3 colours in a single composition
  • Use semantic colours purely for decoration
  • Use unapproved tints or custom shades
  • Place light text on Amber — insufficient contrast

Monospace — DM Mono
DM Mono is reserved for technical content — code snippets, file names, hex values, and data references. Never use it for body copy.
const brand = {
  primaryColor: "#3B6EF5",
  font: "DM Sans",
  logoFile: "Logo_Primary_Dark.svg"
}
Typography Rules
✓ Do
  • Use DM Sans for all brand communications
  • Use ExtraBold (800) for display headings only
  • Maintain a clear type hierarchy at all times
  • Use DM Mono for code, filenames, and data
✗ Don't
  • Use more than 2 typefaces in a single design
  • Set body text smaller than 13px
  • Use bold weight for long passages of text
  • Use DM Mono for decorative or headline purposes
Photography Style
Acme photography is clean, purposeful, and human. We favour natural, candid moments over heavily staged or stock-like imagery.
Illustration Style
When illustrations are used, they follow a clean geometric line-art style consistent with the brand's precision and clarity. Always use brand colours.
Photography Rules
✓ Do
  • Use natural lighting and real environments
  • Show diverse, authentic people in genuine moments
  • Keep compositions clean and intentional
  • Maintain consistent colour temperature across a campaign
✗ Don't
  • Use obviously staged or generic stock photography
  • Apply heavy filters or dramatic colour grading
  • Use imagery featuring competitor brand logos
  • Use low-resolution or compressed imagery
Our Brand Voice
Acme speaks with clarity, confidence, and warmth. We are genuine experts who never talk down to our audience.
Voice vs. Tone
Our voice is who we are — it never changes. Our tone adapts to the situation, just like a person adjusts how they speak depending on context.
Our voice is always:
  • Confident but not boastful
  • Clear but not blunt
  • Human but not unprofessional
  • Expert but not exclusive
Our tone shifts by context:
  • Onboarding → warm & encouraging
  • Errors → calm & reassuring
  • Marketing → energetic & bold
  • Support docs → precise & helpful

Writing Rules
✓ Do
  • Write in active voice: "Upload your logo" not "Logo can be uploaded"
  • Use contractions to feel natural: "you're", "we'll", "it's"
  • Lead with the benefit, follow with the detail
  • Use sentence case for headings — not Title Case For Everything
✗ Don't
  • Use jargon, buzzwords, or corporate-speak
  • Write passive voice when active is possible
  • Use exclamation marks excessively!
  • Be vague: "improved performance" — say how much
Icon Style
Acme uses a consistent outlined icon style based on a 24×24px grid with 2px stroke weight and rounded line caps. All icons come from a unified system to ensure visual coherence.
Set Lucide Icons
Stroke 2px
Grid 24×24px
Style Outlined, rounded
Sizing System
Icons should only be used at the following sizes to maintain visual consistency across all touchpoints.
Icon Usage Rules
✓ Do
  • Use icons at approved sizes: 13, 16, 20, or 24px
  • Pair icons with text labels wherever possible
  • Use Acme Blue for interactive or active icons
  • Maintain consistent 2px stroke weight
✗ Don't
  • Mix outline and filled icon styles in the same UI
  • Scale icons to arbitrary or fractional sizes
  • Use icons as the sole indicator of meaning (add labels)
  • Use icons from other icon systems alongside ours
