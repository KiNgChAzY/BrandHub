Colour Usage Rules
✓ Do
  • Use Acme Blue as the primary CTA and interactive colour
  • Use Deep Navy for headlines on light backgrounds
  • Reserve semantic colours (green, amber, red) for system states
  • Ensure 4.5:1 contrast ratio for text on coloured backgrounds
✗ Don't
  • Mix more than 3 colours in a single composition
  • Use semantic colours purely for decoration
  • Use unapproved tints or custom shades
  • Place light text on Amber — insufficient contrast

Monospace — DM Mono
DM Mono is reserved for technical content — code snippets, file names, hex values, and data references. Never use it for body copy.
const brand = {
  primaryColor: "#3B6EF5",
  font: "DM Sans",

Typography Rules
✓ Do
  • Use DM Sans for all brand communications
  • Use ExtraBold (800) for display headings only
  • Maintain a clear type hierarchy at all times
  • Use DM Mono for code, filenames, and data
✗ Don't
  • Use more than 2 typefaces in a single design
  • Set body text smaller than 13px
  • Use bold weight for long passages of text
  • Use DM Mono for decorative or headline purposes

Our Brand Voice
Acme speaks with clarity, confidence, and warmth. We are genuine experts who never talk down to our audience. Our voice stays consistent — our tone adapts to context.
Confident
We speak with authority backed by real expertise. Not arrogant — assured. We know our craft and stand behind what we build.
Clear
We cut through jargon and complexity. We say exactly what we mean in plain, direct language. No fluff, no filler.
Human
We're real people talking to real people. Warm, approachable, and genuinely helpful — without being casual to the point of feeling unprofessional.
Voice vs. Tone
Our voice is who we are — it never changes. Our tone adapts to the situation, just like a person adjusts how they speak depending on context.
Our voice is always:
  • Confident but not boastful
  • Clear but not blunt
  • Human but not unprofessional
  • Expert but not exclusive
Our tone shifts by context:
  • Onboarding → warm & encouraging
  • Errors → calm & reassuring
  • Marketing → energetic & bold
  • Support docs → precise & helpful

Writing Rules
✓ Do
  • Write in active voice: "Upload your logo" not "Logo can be uploaded"
  • Use contractions to feel natural: "you're", "we'll", "it's"
  • Lead with the benefit, follow with the detail
  • Use sentence case for headings — not Title Case For Everything
✗ Don't
  • Use jargon, buzzwords, or corporate-speak
  • Write passive voice when active is possible
  • Use exclamation marks excessively!
  • Be vague: "improved performance" — say how much
Icon Style
Acme uses a consistent outlined icon style based on a 24×24px grid with 2px stroke weight and rounded line caps. All icons come from a unified system to ensure visual coherence.
Sizing System
Icons should only be used at the following sizes to maintain visual consistency across all touchpoints.
13px
Inline / compact
16px

20px

24px

Icon Usage Rules
✓ Do
  • Use icons at approved sizes: 13, 16, 20, or 24px
  • Pair icons with text labels wherever possible
  • Use Acme Blue for interactive or active icons
  • Maintain consistent 2px stroke weight
✗ Don't
  • Mix outline and filled icon styles in the same UI
  • Scale icons to arbitrary or fractional sizes
  • Use icons as the sole indicator of meaning (add labels)
  • Use icons from other icon systems alongside ours
